# 📋 الخطة التطويرية رقم 2 - دمج Plik في مشروع "لا تشيل هم"

> **الحالة:** معلقة - تُنفذ عند طلب المستخدم بكلمة "الخطة التطويرية"

## 📌 ملخص
دمج مميزات نظام Plik (رفع ملفات مؤقتة مثل WeTransfer) في مشروع "لا تشيل هم" القائم.

## ⚠️ ملاحظة مهمة
- تحميل الفيديو بدون علامة مائية (الوظيفة الأصلية) **لن يتأثر إطلاقاً**
- كل المراحل التالية هي **إضافات جديدة** وليست تعديلات على الكود الحالي
- المشروع سيصبح نظام متكامل: تحميل فيديوهات + رفع ملفات

## 📐 المراحل

### المرحلة 1: نظام رفع الملفات (الأساس)
- مكون رفع ملفات بسحب وإفلات + اختيار من الجهاز
- شريط تقدم للرفع مع إمكانية الإلغاء
- API routes لرفع/تحميل الملفات
- تخزين مؤقت (In-memory + filesystem)
- روابط مشاركة مع ID فريد وتاريخ انتهاء
- دعم ملفات متعددة في رفعة واحدة

### المرحلة 2: صفحة التحميل والمعاينة
- صفحة تحميل بـ ID الملف
- معاينة الملفات: صور، فيديو، PDF، نصوص
- زر نسخ الرابط ومشاركة
- حماية بكلمة مرور (اختياري)

### المرحلة 3: لوحة إدارة المستخدم
- إحصائيات: عدد الرفعات، حجم الإجمالي، الملفات
- سجل الرفعات مع تصفية وترتيب
- إدارة الملفات: حذف، تمديد مدة الصلاحية
- تصدير/استيراد السجل

### المرحلة 4: نظام المصادقة
- تسجيل/دخول محلي (إيميل + كلمة مرور)
- حفظ الجلسة في cookies
- API Tokens للوصول البرمجي
- حصص المستخدم (حجم أقصى، عدد ملفات)

### المرحلة 5: مميزات متقدمة
- OneShot: حذف بعد أول تحميل
- TTL قابل للتخصيص: ساعة، يوم، أسبوع، شهر
- كلمة مرور للملفات الحساسة
- سحب وإفلات متعدد الملفات
- شارات (Badges) للرفعات

### المرحلة 6: التحسينات النهائية
- توحيد التصميم مع Plik (glass-card style)
- إضافة الترجمات العربية/الإنجليزية لجميع المكونات الجديدة
- تحسين الأداء والتجاوب
- اختبار شامل
